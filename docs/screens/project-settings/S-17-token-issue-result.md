## 状態バリエーション

### ステップ遷移

| 状態 | 表示内容 | 操作 |
|---|---|---|
| ステップ 1（初期） | トークン名入力フォーム | 発行 or キャンセル |
| リクエスト中 | ステップ 1 のまま、発行ボタン disabled + スピナー | 待機 |
| ステップ 2（発行成功） | 平文トークン + 警告 + 閉じるボタン | コピー or 閉じる |

### コピーボタンの状態

| 状態 | 表示 |
|---|---|
| 初期 | 「コピー」 |
| コピー成功後 | 「コピーしました ✓」（2 秒後に「コピー」に戻る） |

### ダイアログ閉じ防止

- ステップ 2 ではオーバーレイ背景クリック・Escape キーでの閉じを**無効化**する（平文を確認せずに閉じることを防ぐ）
- 閉じるボタンのクリックのみでダイアログを閉じる

## API 呼び出し

| トリガー | メソッド | パス | リクエスト | レスポンス処理 |
|---|---|---|---|---|
| 発行ボタンクリック | POST | `/api/v1/projects/:pid/tokens` | `{ "name": "discovery-ci" }` | ステップ 2 に遷移し平文を表示 |

### リクエスト

```json
{
  "name": "discovery-satellite-prod"
}
```

### レスポンス 201

```json
{
  "id": "uuid",
  "name": "discovery-satellite-prod",
  "token": "tms_base64url-encoded-32bytes-or-more",
  "created_at": 1719388800000
}
```

**レスポンスヘッダ:**
```
Cache-Control: no-store
```

- `token` フィールドは発行レスポンスのみに含まれる

## エラー状態

| エラー | HTTP | 表示方法 |
|---|---|---|
| トークン名が空 | — | フォームバリデーション。発行ボタン disabled |
| 認証切れ | 401 | ログイン画面（S-02）にリダイレクト |
| ネットワークエラー | — | ダイアログ内にインラインエラー「トークンの発行に失敗しました」 |

**実装済みの事実:** トークン名の重複チェックは行わない（`api_tokens` テーブルに `name` の一意制約は無く、`POST /api/v1/projects/:pid/tokens` も重複を検出しない）。同一プロジェクト内で同名のトークンを複数発行できる。
