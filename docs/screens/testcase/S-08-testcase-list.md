## 要素カタログ

### ヘッダー領域

| 要素名 | 種別 | data-testid | 必須/任意 | 備考 |
|---|---|---|---|---|
| プロジェクト名 | テキスト表示 | `project-name` | — | 現在のプロジェクト名。クリックで S-06 に戻る |
| テストケース件数 | テキスト表示 | `testcase-count` | — | フィルタ適用後の該当件数を括弧表示 |
| 新規作成ボタン | ボタン | `btn-create-testcase` | — | editor+ のみ表示。クリックで S-09 に遷移 |
| 戻るリンク | リンク | `link-back-to-project` | — | `← プロジェクト名` 形式。S-06 に遷移 |

### フィルタパネル

| 要素名 | 種別 | data-testid | 選択肢 | 既定値 | 備考 |
|---|---|---|---|---|---|
| ステータス | ドロップダウン | `filter-status` | `all` / `draft` / `approved` / `archived` | `all` | API: `status` パラメータ |
| カテゴリ | ドロップダウン | `filter-category` | `all` / `normal` / `abnormal` / `boundary` / `error_handling` | `all` | API: `category` パラメータ |
| 所有権 | ドロップダウン | `filter-ownership` | `all` / `machine` / `human` | `all` | API: `ownership` パラメータ |
| drift | ドロップダウン | `filter-drift` | `all` / `true` / `false` | `all` | API: `drift` パラメータ（boolean） |
| stale | ドロップダウン | `filter-stale` | `all` / `true` / `false` | `all` | API: `is_stale` パラメータ（boolean） |
| target | テキスト入力 | `filter-target` | — | 空 | 部分一致検索。API: `target` パラメータ |
| フリーテキスト検索 | テキスト入力 | `filter-search` | — | 空 | `[※]` タイトル・Given/When/Then の全文検索 |
| フィルタクリアボタン | ボタン | `btn-clear-filters` | — | — | 全フィルタを既定値にリセット |

フィルタ変更時の挙動:
- 変更の都度、API を再リクエスト（デバウンス: テキスト入力は 300ms）
- カーソルをリセットして先頭ページから表示
- URL のクエリパラメータにフィルタ状態を反映（ブラウザバック・共有対応）

## 状態バリエーション

| 状態 | 表示変化 |
|---|---|
| viewer ロール | 新規作成ボタン・チェックボックス・一括操作バー非表示 |
| editor+ ロール | 全要素表示 |
| テストケース 0 件（フィルタなし） | 空状態メッセージ + 新規作成誘導 |
| テストケース 0 件（フィルタあり） | 「該当するテストケースがありません」+ フィルタクリアリンク |
| 同期データなし | 同期サマリーパネル非表示 |
| 同期データあり | 同期サマリーパネル表示 |
| 選択中テストケースあり | 一括操作メニューが enabled |
| 選択中テストケースなし | 一括操作メニューが disabled |
| ダッシュボードからフィルタ付き遷移 | URL クエリパラメータでフィルタ初期値を設定 |

## API コール

| トリガー | メソッド | パス | パラメータ/ヘッダ | レスポンス処理 |
|---|---|---|---|---|
| 画面表示・フィルタ変更・ページ遷移 | GET | `/api/v1/projects/:pid/testcases` | `status`, `category`, `ownership`, `drift`, `is_stale`, `target`, `cursor`, `limit` | `{ items, next_cursor, has_more }` でテーブル描画 |
| 一括操作実行（S-15 確認後） | POST | `/api/v1/projects/:pid/testcases/bulk` | `{ ids: [...], action: "approve" \| "archive" \| "restore" }` | `{ updated, skipped, errors }` — 完了後にテーブル再取得 |

## エラー状態

| エラー | HTTP | コード | 表示 |
|---|---|---|---|
| 認証切れ | 401 | `UNAUTHORIZED` | ログイン画面（S-02）にリダイレクト |
| プロジェクト未存在/権限なし | 404 | `NOT_FOUND` | 「プロジェクトが見つかりません」エラーページ |
| 一括操作の部分失敗 | 200（本体内に errors 配列） | `OCC_CONFLICT` 等 | トースト「N件更新、M件スキップ、K件エラー」 |
| ネットワークエラー | — | — | トースト「接続エラー。再試行してください」 |
| レート制限 | 429 | `RATE_LIMITED` | トースト「リクエストが多すぎます。しばらく待ってください」 |
