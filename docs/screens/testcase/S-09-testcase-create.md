## 要素カタログ

### 基本フィールド

| 要素名 | 種別 | data-testid | 必須 | バリデーション | 備考 |
|---|---|---|---|---|---|
| タイトル | テキスト入力 | `input-title` | yes | 空でない。最大長は実装時に決定 | テストの意図を簡潔に記述 |
| 対象 (target) | テキスト入力 | `input-target` | no | 自由入力。フォーマット制約なし | 例: `com.example.PaymentService#charge` |
| 対象ヘルプテキスト | テキスト表示 | `hint-target` | — | — | 「例: com.example.PaymentService#charge」 |
| カテゴリ | ドロップダウン | `select-category` | yes | `normal` / `abnormal` / `boundary` / `error_handling` | 既定値: `normal` |

### Given/When/Then

| 要素名 | 種別 | data-testid | 必須 | バリデーション | 備考 |
|---|---|---|---|---|---|
| Given テキストエリア | テキストエリア | `textarea-given` | yes | 空でない | 事前条件。複数行可 |
| When テキストエリア | テキストエリア | `textarea-when` | yes | 空でない | 操作。複数行可 |
| Then テキストエリア | テキストエリア | `textarea-then` | yes | 空でない | 期待結果。複数行可 |

### パラメータテーブル（動的行）

| 要素名 | 種別 | data-testid | 必須 | バリデーション | 備考 |
|---|---|---|---|---|---|
| パラメータセクション | コンテナ | `section-parameters` | no | — | 折りたたみ可能（任意セクション） |
| パラメータ行 | 行 | `param-row-{index}` | — | — | index は 0 始まり |
| name 入力 | テキスト入力 | `param-name-{index}` | no | — | パターン名。例: 「負の値」 |
| inputs 入力 | テキスト入力 | `param-inputs-{index}` | yes（行が存在する場合） | JSON 形式。構文チェック | 入力値。例: `{"age": -1}` |
| expected 入力 | テキスト入力 | `param-expected-{index}` | yes（行が存在する場合） | 空でない | 期待結果。例: 「エラー」 |
| 行削除ボタン | ボタン | `btn-param-remove-{index}` | — | — | 該当行を削除 |
| 行追加ボタン | ボタン | `btn-param-add` | — | — | 末尾に空行を追加 |

### メタデータ

| 要素名 | 種別 | data-testid | 必須 | バリデーション | 備考 |
|---|---|---|---|---|---|
| メタデータセクション | コンテナ | `section-metadata` | no | — | 折りたたみ可能 |
| タグトークン入力 | トークン入力 | `input-tags` | no | 各タグ: 空でない文字列 | Enter キーまたは追加ボタンでタグ確定 |
| タグバッジ | バッジ | `tag-{value}` | — | — | 各タグの表示。× ボタンで削除可 |
| タグ追加ボタン | ボタン | `btn-tag-add` | — | — | 入力中のテキストをタグとして追加 |

### アクションボタン

| 要素名 | 種別 | data-testid | 備考 |
|---|---|---|---|
| キャンセルボタン | ボタン | `btn-cancel` | 確認ダイアログなしで S-08 に戻る（入力内容は破棄） |
| 下書き保存ボタン | ボタン | `btn-save-draft` | `status: "draft"` で作成。バリデーション通過が必要 |
| 作成ボタン | ボタン（プライマリ） | `btn-create` | `status: "draft"` で作成。下書き保存と同義（MVP では区別なし） |

### フォームバリデーションエラー

| 要素名 | 種別 | data-testid | 備考 |
|---|---|---|---|
| フィールドエラー | エラーテキスト | `error-{field}` | 各フィールド直下に赤文字表示。field = `title`, `given`, `when`, `then` 等 |
| フォーム全体エラー | エラーバナー | `form-error` | サーバーサイドエラー時にフォーム上部に表示 |

## 状態バリエーション

| 状態 | 表示変化 |
|---|---|
| 初期表示 | 全フィールド空。カテゴリのみ `normal` が選択済み |
| バリデーションエラー | 必須フィールド未入力時、該当フィールドに赤枠 + エラーメッセージ表示 |
| 送信中 | 作成ボタン disabled + ローディングスピナー表示 |
| パラメータ行なし | パラメータセクションに「+ 行を追加」ボタンのみ表示 |
| パラメータ行あり | テーブル形式で表示。各行に削除ボタン |
| タグなし | タグ入力フィールドのみ表示 |
| タグあり | タグバッジが横並びで表示 |

## API コール

| トリガー | メソッド | パス | リクエストボディ | レスポンス処理 |
|---|---|---|---|---|
| 作成ボタン押下 | POST | `/api/v1/projects/:pid/testcases` | `{ title, target?, category, given, when, then, parameters?, status: "draft", metadata? }` | 201 → S-10 にリダイレクト（作成されたテストケース ID） |

リクエストボディ詳細:

```json
{
  "title": "有効期限切れカードで決済を試みるとエラーが返る",
  "target": "com.example.PaymentService#charge",
  "category": "abnormal",
  "given": "有効期限が過去のカード情報が登録されている",
  "when": "そのカードで 1,000 円の決済を実行する",
  "then": "決済が拒否され、エラーコード CARD_EXPIRED が返る",
  "parameters": [
    { "name": "期限切れ", "inputs": { "card_expiry": "2020-01" }, "expected": "CARD_EXPIRED" }
  ],
  "status": "draft",
  "metadata": { "tags": ["payment", "card-validation"] }
}
```

作成後のテストケース:
- `ownership`: `human`（手動作成のため）
- `created_origin`: `manual`

## エラー状態

| エラー | HTTP | コード | 表示 |
|---|---|---|---|
| バリデーションエラー（クライアント） | — | — | フィールド直下にインラインエラー表示 |
| バリデーションエラー（サーバー） | 422 | `VALIDATION_FAILED` | `details[].path` に基づき該当フィールドにエラー表示 |
| 認証切れ | 401 | `UNAUTHORIZED` | ログイン画面にリダイレクト |
| 権限不足 | 403 | — | viewer がアクセスした場合。S-08 にリダイレクト |
| ネットワークエラー | — | — | フォーム上部にエラーバナー「保存に失敗しました。再試行してください」 |
