## 状態バリエーション

| 状態 | 表示変化 |
|---|---|
| ownership=machine で編集ボタン押下 | 先に machine 警告ダイアログを表示。「編集を続ける」で編集モードに入る |
| ownership=human で編集ボタン押下 | 直接編集モードに入る（警告なし） |
| 編集中（未変更） | 保存ボタンは有効（サーバーが no-op を判定） |
| 編集中（変更あり） | 保存ボタン有効 |
| 保存送信中 | 保存ボタン disabled + ローディングスピナー |
| OCC 競合発生 | 競合エラーバナー + 再読み込みボタン表示。フォームは編集可能のまま |
| バリデーションエラー | 該当フィールドに赤枠 + エラーメッセージ |

## API コール

| トリガー | メソッド | パス | ヘッダ/ボディ | レスポンス処理 |
|---|---|---|---|---|
| 保存ボタン押下 | PATCH | `/api/v1/projects/:pid/testcases/:id` | `If-Match: "<version>"`, body: 変更フィールドのみ | 200 → S-10 閲覧モードに戻る。新 ETag を保持 |

### PATCH セマンティクス

送信ボディには変更があったフィールドのみを含む:

```json
{
  "title": "新しいタイトル",
  "then": "更新された期待結果",
  "category": "abnormal"
}
```

- キーが未指定 → そのフィールドは不変
- キーに明示的 `null` → そのフィールドをクリア（target 等の任意フィールド）

### ownership 遷移

`ownership=machine` のテストケースに対して、人間所有列の値が**実際に変化する** PATCH を行うと:
1. `ownership` が `machine → human` に不可逆遷移する
2. `version` が +1 される
3. 以後 Discovery の自動更新（ミラー昇格）は停止する
4. コード側の変化は `drift` として検知されるようになる

同値 PATCH（フィールドの値が変わらない場合）では遷移しない。

## 画面遷移

| アクション | 遷移先 | 備考 |
|---|---|---|
| 保存成功 | S-10 閲覧モード | 同一ページ。フォーム → 表示に切替 |
| キャンセル押下 | S-10 閲覧モード | 編集内容破棄 |
| machine 警告でキャンセル | S-10 閲覧モード | 編集モードに入らない |
| OCC 競合 → 再読み込み | S-11 編集モード（最新データで再描画） | フォームに最新の値がプリフィルされる |

## エラー状態

| エラー | HTTP | コード | 表示 |
|---|---|---|---|
| OCC 競合 | 409 | `OCC_CONFLICT` | 競合エラーバナー「他のユーザーが先に更新しました。最新の内容を確認してください」+ 再読み込みボタン |
| If-Match 未指定 | 428 | `PRECONDITION_REQUIRED` | クライアントバグ（通常発生しない） |
| バリデーションエラー（サーバー） | 422 | `VALIDATION_FAILED` | `details[].path` に基づきフィールドエラー表示 |
| 認証切れ | 401 | `UNAUTHORIZED` | ログイン画面にリダイレクト |
