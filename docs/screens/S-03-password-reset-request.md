## 要素カタログ

### ページヘッダー

| 要素名 | 型 | data-testid | 必須 | 説明 |
|---|---|---|---|---|
| ページタイトル | text-display | `password-reset-title` | — | 「パスワードリセット」固定テキスト |
| 説明テキスト | text-display | `password-reset-description` | — | リセット手順の説明 |

### フォーム入力（送信前状態）

| 要素名 | 型 | data-testid | 必須 | バリデーション | 備考 |
|---|---|---|---|---|---|
| メールアドレス | email-input | `password-reset-email` | yes | 空文字不可、メールアドレス形式 | type="email" |

### アクションボタン / リンク

| 要素名 | 型 | data-testid | 条件 | 動作 |
|---|---|---|---|---|
| 送信ボタン | button (primary) | `password-reset-submit` | メールアドレス入力済み時に有効 | API 呼び出し → 完了画面に切替 |
| ログインに戻る | link | `password-reset-back-to-login` | 常時表示 | S-02 に遷移 |

### 送信完了メッセージ（送信後状態）

| 要素名 | 型 | data-testid | 表示条件 | 説明 |
|---|---|---|---|---|
| 完了メッセージ | alert (success) | `password-reset-success` | API 送信成功後 | 「リセット用のリンクを送信しました」 |
| 補足テキスト | text-display | `password-reset-success-hint` | API 送信成功後 | 「メールをご確認ください」 |

## 状態バリエーション

| 状態 | 条件 | 画面の変化 |
|---|---|---|
| 初期状態 | フォーム未入力 | メール入力フィールドと送信ボタン表示 |
| 入力中 | メールアドレス入力済み | 送信ボタン enabled |
| 送信中 | API リクエスト中 | 送信ボタン loading 状態 |
| 送信完了 | API レスポンス受信後 | フォームが完了メッセージに切替。メールアドレスの存在有無に関わらず同一表示 |

**セキュリティ要件:**
- メールアドレスの存在有無に関わらず常に同一のレスポンス・UI を返す
- タイミング攻撃防止のため、存在しないメールでも同等の処理時間をかける

---

## API 呼び出し

### フォーム送信時

| メソッド | パス | トリガー | リクエストボディ |
|---|---|---|---|
| POST | `/api/v1/auth/password-reset-request` | 送信ボタンクリック | `{ email }` |

**レスポンス処理:**

| ステータス | 処理 |
|---|---|
| 200 | 完了メッセージを表示（常に成功表示） |

> API 自体が `[※未実装]`。実装時はメールアドレスの存在有無に関わらず同一レスポンスを返す設計。

## エラー状態

### フィールドバリデーション（クライアント側）

| フィールド | エラー条件 | エラーメッセージ例 |
|---|---|---|
| メールアドレス | 空文字 | 「メールアドレスを入力してください」 |

### API エラー

> セキュリティ上、API エラーは基本的にユーザーに表示しない。ネットワークエラー等の場合のみ汎用エラーを表示。

| 状況 | エラーメッセージ例 |
|---|---|
| ネットワークエラー | 「通信エラーが発生しました。再度お試しください」 |
