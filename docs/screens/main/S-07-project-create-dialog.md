# S-07 プロジェクト作成ダイアログ

| 項目 | 内容 |
|---|---|
| **画面 ID** | S-07 |
| **画面名** | プロジェクト作成ダイアログ |
| **種別** | ダイアログ（S-06 上にオーバーレイ） |
| **URL** | なし（S-06 上のモーダル） |
| **対応 UC** | UC-04 |
| **必要ロール** | admin |
| **スペック状況** | 定義済み（一覧内） |

## 概要

新しいプロジェクトを作成するためのモーダルダイアログ。S-06 プロジェクト一覧の「新規作成」ボタンから開く。作成完了後はダイアログが閉じ、S-06 が最新状態に更新される。

---

## ワイヤーフレーム

```
┌──────────────────────────────────┐
│ プロジェクト作成            [✕]  │
│                                  │
│ プロジェクト名 [______________]  │
│ リポジトリ URL [______________]  │
│               (任意)             │
│                                  │
│        [キャンセル] [作成]       │
└──────────────────────────────────┘
```

## フォームバリデーション

### クライアントサイド

| フィールド | ルール | エラーメッセージ |
|---|---|---|
| プロジェクト名 | 空文字不可 | 「プロジェクト名を入力してください」 |
| プロジェクト名 | 最大長制限（実装依存） | 「プロジェクト名が長すぎます」 |
| リポジトリ URL | 入力時は URL 形式 | 「有効な URL を入力してください」 |

### サーバーサイド

| エラーコード | 条件 | 表示 |
|---|---|---|
| `VALIDATION_FAILED` (422) | バリデーション失敗 | 該当フィールドに `details[].msg` を表示 |

**実装済みの事実:** プロジェクト名の重複チェックは行わない(`projects` テーブルに `name` の一意制約は無く、`POST /api/v1/projects` も重複を検出しない)。同一組織内で同名のプロジェクトを複数作成できる。

---

## 状態バリエーション

| 状態 | 表示内容 |
|---|---|
| 初期状態 | フォーム空欄。作成ボタン有効 |
| 入力中 | フォームに値あり。作成ボタン有効 |
| バリデーションエラー | エラーメッセージ表示。作成ボタン有効 |
| 送信中 | 作成ボタン disabled + スピナー。フォーム入力も disabled |
| サーバーエラー | エラーメッセージ表示。フォーム・ボタン再有効化 |

## API 呼び出し

| メソッド | パス | トリガー | レスポンス処理 |
|---|---|---|---|
| POST | `/api/v1/projects` | 作成ボタンクリック | 201: ダイアログを閉じ S-06 を再取得。トースト表示 |

**リクエスト:**
```json
{
  "name": "payment-service",
  "repo_url": "https://github.com/example/payment"
}
```

**レスポンス 201:**
```json
{
  "id": "uuid",
  "name": "payment-service",
  "repo_url": "https://github.com/example/payment",
  "created_at": 1719388800000,
  "updated_at": 1719388800000
}
```

## 画面遷移

| アクション | 遷移先 | 条件 |
|---|---|---|
| 作成完了 | S-06 プロジェクト一覧（ダイアログ閉じ + 一覧リロード） | — |
| キャンセルボタン | S-06 プロジェクト一覧（ダイアログ閉じ） | — |
| ✕ ボタン | S-06 プロジェクト一覧（ダイアログ閉じ） | — |
| Escape キー | S-06 プロジェクト一覧（ダイアログ閉じ） | — |

---

## エラー状態

| エラー | HTTP | 表示 |
|---|---|---|
| バリデーションエラー | 422 | フィールドごとのインラインエラーメッセージ |
| 未認証 | 401 | S-02 ログインへリダイレクト |
| 権限不足 | 403 | エラートースト「この操作を行う権限がありません」 |
| サーバーエラー | 500 | エラートースト「プロジェクトの作成に失敗しました」 |

| キー | 動作 |
