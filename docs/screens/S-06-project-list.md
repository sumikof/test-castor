# S-06 プロジェクト一覧

| 項目 | 内容 |
|---|---|
| **画面 ID** | S-06 |
| **画面名** | プロジェクト一覧 |
| **種別** | ページ |
| **URL** | `/projects` |
| **対応 UC** | UC-04, UC-29 |
| **必要ロール** | viewer 以上（全認証済みユーザー） |
| **スペック状況** | 定義済み |

## 概要

組織内のプロジェクトを一覧表示し、新規プロジェクトの作成（admin のみ）を行う画面。各プロジェクト行をクリックするとそのプロジェクトのテストケース一覧（S-08）に遷移する。

## 状態バリエーション

| 状態 | 表示内容 |
|---|---|
| admin / プロジェクト 0 件 | 空状態メッセージ + 新規作成ボタン表示 |
| admin / プロジェクトあり | テーブル表示 + 新規作成ボタン表示 |
| editor・viewer / プロジェクト 0 件 | 空状態メッセージのみ（作成ボタンなし） |
| editor・viewer / プロジェクトあり | テーブル表示のみ（作成ボタンなし） |
| ローディング中 | テーブル部分にスケルトン/スピナー |

## API 呼び出し

| メソッド | パス | トリガー | レスポンス処理 |
|---|---|---|---|
| GET | `/api/v1/projects` | 画面表示時 | `items` 配列をテーブルに表示 |

**レスポンス例:**
```json
{
  "items": [
    {
      "id": "uuid",
      "name": "payment-service",
      "repo_url": "https://github.com/example/payment",
      "created_at": 1719388800000,
      "updated_at": 1719388800000
    }
  ]
}
```

> **注:** テスト数は `GET /api/v1/projects` のレスポンスに含まれていない。実装時に以下のいずれかの対応が必要:
> - プロジェクト API のレスポンスに `testcase_count` フィールドを追加する
> - プロジェクトごとに `GET /api/v1/projects/:pid/testcases?limit=0` でカウントを取得する
> - テスト数列を表示しない

## 画面遷移

| アクション | 遷移先 | 条件 |
|---|---|---|
| プロジェクト行クリック | S-08 テストケース一覧（`/projects/:pid/testcases`） | — |
| 新規作成ボタン | S-07 プロジェクト作成ダイアログ（オーバーレイ） | admin のみ |
| ヘッダー: ダッシュボード | S-05 ダッシュボード | — |
| ヘッダー: ユーザー管理 | S-18 ユーザー一覧 | admin のみ |
| ヘッダー: プロフィール | S-20 プロフィール | — |
| ヘッダー: ログアウト | S-02 ログイン | — |
