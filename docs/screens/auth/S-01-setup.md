# S-01 初期セットアップ

| 項目 | 値 |
|---|---|
| ID | S-01 |
| 画面名 | 初期セットアップ |
| 種別 | ページ |
| URL | `/setup` |
| 対応 UC | UC-02 |
| 認証 | 不要 |
| 必要ロール | なし（Organization 未存在時のみアクセス可） |

## 状態バリエーション

| 状態 | 条件 | 画面の変化 |
|---|---|---|
| 初期表示（SSR・no-JS） | 全フィールド空 | 送信ボタンは disabled にしない（JS 無効でもフォーム送信可能。サーバー側検証が権威） |
| 初期状態（JS 有効時） | 全フィールド空 | Progressive Enhancement: JS がここで初めてボタンに disabled を適用する |
| 入力中（JS 有効時） | 一部フィールドに入力あり | バリデーションエラーはフォーカスアウト時に表示。全フィールド入力済み・バリデーション通過で送信ボタンが enabled になる |
| 送信中（JS 有効時） | API リクエスト中 | 送信ボタンが loading 状態（disabled + スピナー） |
| 成功 | 201 レスポンス | `/login` にリダイレクト |
| エラー（409） | Organization 既存 | エラーメッセージ表示 + `/login` にリダイレクト |
| エラー（422） | バリデーション失敗 | 該当フィールドにインラインエラー表示 |
| リダイレクト済み | Organization が既に存在 | `/login` に自動リダイレクト（画面表示されない） |

---

## API 呼び出し

### ページロード時

| メソッド | パス | トリガー | レスポンス処理 |
|---|---|---|---|
| GET | `/api/v1/setup` または Organization 存在チェック | 画面マウント時 | Organization が存在すれば `/login` にリダイレクト |

### フォーム送信時

| メソッド | パス | トリガー | リクエストボディ |
|---|---|---|---|
| POST | `/api/v1/setup` | 送信ボタンクリック | `{ organization_name, admin_email, admin_password, admin_display_name }` |

**レスポンス処理:**

| ステータス | 処理 |
|---|---|
| 201 | 成功トースト → `/login` にリダイレクト |
| 409 `SETUP_ALREADY_COMPLETE` | エラーメッセージ表示 → `/login` にリダイレクト |
| 422 `VALIDATION_FAILED` | `details[].path` に基づきフィールド単位のインラインエラー表示 |

|---|---|---|

## エラー状態

### フィールドバリデーション（クライアント側）

| フィールド | エラー条件 | エラーメッセージ例 |
|---|---|---|
| 組織名 | 空文字 | 「組織名を入力してください」 |
| 管理者メール | 空文字 | 「メールアドレスを入力してください」 |
| 管理者メール | 不正な形式 | 「有効なメールアドレスを入力してください」 |
| パスワード | 空文字 | 「パスワードを入力してください」 |
| パスワード | ポリシー不適合 | 「パスワードは8文字以上で入力してください」 |
| パスワード確認 | パスワードと不一致 | 「パスワードが一致しません」 |
| 表示名 | 空文字 | 「表示名を入力してください」 |

### API エラー

| コード | HTTP | エラーメッセージ例 |
|---|---|---|
| `SETUP_ALREADY_COMPLETE` | 409 | 「セットアップは既に完了しています」 |
| `VALIDATION_FAILED` | 422 | フィールドごとのインラインエラー |
