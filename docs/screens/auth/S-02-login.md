# S-02 ログイン

| 項目 | 値 |
|---|---|
| ID | S-02 |
| 画面名 | ログイン |
| 種別 | ページ |
| URL | `/login` |
| 対応 UC | UC-03 |
| 認証 | 不要 |
| 必要ロール | なし |
| スペック状況 | API定義済み |

## 表示条件

- 未認証ユーザーがアクセスする認証画面
- 認証済みの場合はダッシュボード（S-05）またはプロジェクト一覧（S-06）にリダイレクト
- Organization が 0 件の場合は `/setup` (S-01) にリダイレクト

### フォーム入力

| 要素名 | 型 | data-testid | 必須 | バリデーション | 備考 |
|---|---|---|---|---|---|
| メールアドレス | email-input | `login-email` | yes | 空文字不可、メールアドレス形式 | type="email"、autocomplete="email" |

| 要素名 | 型 | data-testid | 条件 | 動作 |
|---|---|---|---|---|

|---|---|---|---|---|

## 状態バリエーション

| 状態 | 条件 | 画面の変化 |
|---|---|---|
| 初期表示（SSR・no-JS） | フィールド空 | ログインボタンは disabled にしない（JS 無効でもフォーム送信可能。サーバー側検証が権威） |
| 初期状態（JS 有効時） | フィールド空 | Progressive Enhancement: JS がここで初めてボタンに disabled を適用する |
| 入力中（JS 有効時） | 一部入力済み | ログインボタンは両フィールド入力済みで enabled |
| 送信中（JS 有効時） | API リクエスト中 | ログインボタン loading 状態（disabled + スピナー） |
| 認証失敗 | 401 レスポンス | エラーバナー表示、パスワードフィールドクリア |
| レート制限 | 429 レスポンス | エラーバナー表示（リトライまでの待機案内） |
| 成功 | 200 レスポンス + Set-Cookie | リダイレクト実行 |
| セットアップ完了後遷移 | S-01 からのリダイレクト | 成功トースト「セットアップが完了しました」が表示されている |

## API 呼び出し

### ページロード時

| メソッド | パス | トリガー | レスポンス処理 |
|---|---|---|---|
| GET | `/api/v1/auth/me` | 画面マウント時 | 認証済みなら `/projects`（S-06）にリダイレクト |

### フォーム送信時

| メソッド | パス | トリガー | リクエストボディ |
|---|---|---|---|
| POST | `/api/v1/auth/login` | ログインボタンクリック | `{ email, password }` |

**レスポンス処理:**

| ステータス | 処理 |
|---|---|
| 200 | Set-Cookie でセッション発行 → `/projects`（S-06）にリダイレクト。S-05 実装後はそちらに変更 |
| 401 `UNAUTHORIZED` | エラーバナー「メールアドレスまたはパスワードが正しくありません」（統一メッセージ） |
| 429 `RATE_LIMITED` | エラーバナー「ログイン試行回数の上限に達しました。しばらくしてから再度お試しください」 |

**セキュリティ要件:**
- 認証失敗メッセージはメールアドレスの存在有無を漏洩しない統一表現を使用
- ログイン成功時にセッション ID を必ず再発行（セッション固定攻撃対策）
- `(account, IP)` 別の永続カウンタでブルートフォースを防御

## 画面遷移

| アクション | 遷移先 | 条件 |
|---|---|---|
| ログイン成功 | S-06 プロジェクト一覧 (`/projects`) | 200 成功時（S-05 実装前） |
| ログイン成功（将来） | S-05 ダッシュボード (`/dashboard`) | S-05 実装後 |
| 認証済みリダイレクト | S-06 プロジェクト一覧 (`/projects`) | ページロード時に認証セッションが有効 |
| Organization 未存在 | S-01 初期セットアップ (`/setup`) | ページロード時に Organization が 0 件 |

## エラー状態

### フィールドバリデーション（クライアント側）

| フィールド | エラー条件 | エラーメッセージ例 |
|---|---|---|
| メールアドレス | 空文字 | 「メールアドレスを入力してください」 |
| メールアドレス | 不正な形式 | 「有効なメールアドレスを入力してください」 |
| パスワード | 空文字 | 「パスワードを入力してください」 |

### API エラー

| コード | HTTP | エラーメッセージ例 |
|---|---|---|
| `UNAUTHORIZED` | 401 | 「メールアドレスまたはパスワードが正しくありません」 |
| `RATE_LIMITED` | 429 | 「ログイン試行回数の上限に達しました。しばらくしてから再度お試しください」 |
