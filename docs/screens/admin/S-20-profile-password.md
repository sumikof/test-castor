## 要素カタログ

### プロフィール表示セクション（読み取り専用）

| 要素名 | 種別 | data-testid | 説明 |
|---|---|---|---|
| セクションタイトル | テキスト（h1） | `profile-title` | 「プロフィール」固定 |
| 表示名 | テキスト | `profile-display-name` | 現在のユーザーの `display_name` |
| メールアドレス | テキスト | `profile-email` | 現在のユーザーの `email` |
| ロール | テキスト（バッジ） | `profile-role` | 現在のユーザーの `role`（admin/editor/viewer） |

### パスワード変更セクション

| 要素名 | 種別 | data-testid | 必須 | バリデーション | 表示条件 |
|---|---|---|---|---|---|
| セクション見出し | テキスト（h2） | `password-change-heading` | — | — | 常時 |
| 現在のパスワード | password input | `password-current-input` | yes | 空文字不可 | 常時 |
| 新しいパスワード | password input | `password-new-input` | yes | パスワードポリシー準拠（最低 8 文字等） | 常時 |
| パスワード確認 | password input | `password-confirm-input` | yes | `password-new-input` と一致すること | 常時 |
| パスワード変更ボタン | button | `password-change-submit` | — | 全 3 フィールド入力済み + 確認一致で enabled | 常時 |

## 状態バリエーション

### ボタン状態

| 条件 | パスワード変更ボタン |
|---|---|
| いずれかのフィールドが空 | disabled |
| 新しいパスワードと確認が不一致 | disabled |
| 全フィールド入力済み + 一致 | enabled |
| リクエスト中 | disabled + スピナー |

### パスワード確認のインラインバリデーション

- `password-confirm-input` がフォーカスを失った（blur）時点で `password-new-input` と比較
- 不一致の場合:
  - `password-confirm-input` にエラースタイル（赤枠）
  - インラインエラー表示:「パスワードが一致しません」
  - data-testid: `password-confirm-error`

### パスワードポリシーのフィードバック

- `password-new-input` の入力中にポリシー適合状態をリアルタイム表示
- ポリシー項目（各項目に data-testid）:
  - `password-policy-length`: 8 文字以上
  - ※ 追加ポリシー（大文字・数字含有等）は実装時に確定

## API 呼び出し

| トリガー | メソッド | パス | リクエスト | レスポンス処理 |
|---|---|---|---|---|
| ページ読み込み | GET | `/api/v1/auth/me` | — | プロフィール情報を表示 |
| パスワード変更ボタンクリック | PATCH | `/api/v1/auth/password` | 下記参照 | 成功: フォームクリア + トースト |

### GET /api/v1/auth/me レスポンス

```json
{
  "id": "uuid",
  "email": "tanaka@example.com",
  "display_name": "田中太郎",
  "role": "admin",
  "organization_id": "uuid"
}
```

### PATCH /api/v1/auth/password リクエスト

```json
{
  "current_password": "old-password",
  "new_password": "new-secure-password"
}
```

- `password-confirm-input` はクライアント側バリデーションのみ。API には送信しない

### PATCH レスポンス 200

```json
{
  "message": "password_changed"
}
```

### 副作用

- パスワード変更成功時、現在のセッション以外の全セッションが無効化される
- 他のブラウザ・端末からは再ログインが必要になる

## エラー状態

| エラー | HTTP | 表示方法 |
|---|---|---|
| 現在のパスワードが不正 | 401 | インラインエラー「現在のパスワードが正しくありません」（`password-current-input` にエラースタイル）。data-testid: `password-current-error` |
| 新しいパスワードがポリシー不適合 | 422 | インラインエラー「パスワードは 8 文字以上で入力してください」（`password-new-input` にエラースタイル）。data-testid: `password-new-error` |
| パスワード確認不一致 | — | クライアント側バリデーション（上記参照） |
| 認証切れ | 401 | ログイン画面（S-02）にリダイレクト |
| ネットワークエラー | — | エラートースト |
