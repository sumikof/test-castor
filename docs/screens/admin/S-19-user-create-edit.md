# S-19 ユーザー作成・編集ダイアログ

| 項目 | 値 |
|---|---|
| ID | S-19 |
| 画面名 | ユーザー作成・編集 |
| 種別 | ダイアログ（S-18 上のオーバーレイ） |
| URL | なし（S-18 上のモーダル） |
| 対応 UC | UC-05, UC-25 |
| 必要ロール | admin |
| スペック | API定義済み |

## 要素カタログ

### 共通表示要素

| 要素名 | 種別 | data-testid | 説明 |
|---|---|---|---|
| ダイアログタイトル | テキスト（h2） | `user-dialog-title` | 作成:「ユーザー追加」/ 編集:「ユーザー編集: {display_name}」 |

### 作成モードの入力要素

| 要素名 | 種別 | data-testid | 必須 | バリデーション | 表示条件 |
|---|---|---|---|---|---|
| メールアドレス | email input | `user-email-input` | yes | メール形式（RFC 5322）、org 内一意 | 作成モードのみ |
| 表示名 | text input | `user-display-name-input` | yes | 空文字不可、最大 100 文字 | 常時 |
| ロール | select (dropdown) | `user-role-select` | yes | `admin` / `editor` / `viewer` | 常時 |
| 初期パスワード | password input | `user-password-input` | yes | パスワードポリシー準拠（最低 8 文字等） | 作成モードのみ |
| キャンセルボタン | button | `user-dialog-cancel` | — | — | 常時 |
| 追加ボタン | button | `user-dialog-submit-create` | — | 全必須フィールド入力済みで enabled | 作成モードのみ |

### 編集モードの入力・操作要素

| 要素名 | 種別 | data-testid | 必須 | バリデーション | 表示条件 |
|---|---|---|---|---|---|
| メールアドレス（読み取り専用） | テキスト | `user-email-readonly` | — | — | 編集モードのみ。変更不可 |
| 表示名 | text input | `user-display-name-input` | yes | 空文字不可、最大 100 文字 | 常時 |
| ロール | select (dropdown) | `user-role-select` | yes | `admin` / `editor` / `viewer` | 常時 |
| ロール変更警告 | テキスト（alert） | `user-role-change-warning` | — | — | ロールが元の値から変更されたとき表示 |
| パスワードリセットボタン | button | `user-password-reset-button` | — | — | 編集モードのみ |
| ユーザー無効化ボタン | button | `user-disable-button` | — | — | 編集モードのみ `[※未実装]` |
| 保存ボタン | button | `user-dialog-submit-edit` | — | 変更がある場合に enabled | 編集モードのみ |

### パスワードリセット確認ダイアログ（インライン）

| 要素名 | 種別 | data-testid | 説明 |
|---|---|---|---|
| 新しいパスワード入力 | password input | `user-reset-password-input` | 必須。パスワードポリシー準拠 |
| リセットキャンセルボタン | button | `user-reset-password-cancel` | — |
| リセット実行ボタン | button | `user-reset-password-submit` | パスワード入力済みで enabled |

## 状態バリエーション

### モード判定

| 条件 | モード | タイトル | フォーム状態 |
|---|---|---|---|
| S-18 の「追加」ボタンから起動 | 作成 | 「ユーザー追加」 | 全フィールド空 |
| S-18 のユーザー行クリックから起動 | 編集 | 「ユーザー編集: {display_name}」 | 既存データをプリフィル |

### ロール選択肢のデフォルト

| モード | デフォルト値 |
|---|---|
| 作成 | `editor` |
| 編集 | 現在のロール |

### ロール変更時の警告表示

- ロール値が元の値（プリフィル値）と異なる場合に警告テキストを表示
- data-testid: `user-role-change-warning`
- テキスト:「ロール変更時は対象ユーザーの全セッションが無効化されます。次回ログイン時から新しいロールが適用されます」

### 自分自身の編集制限

- 現在ログイン中の admin が自身の行を編集する場合:
  - ロール変更ドロップダウンを disabled にする（自分で自分の admin を剥奪すると管理者不在になるリスク）
  - ツールチップ:「自身のロールは変更できません」

## API 呼び出し

### 作成モード

| トリガー | メソッド | パス | リクエスト | レスポンス処理 |
|---|---|---|---|---|
| 追加ボタンクリック | POST | `/api/v1/users` | 下記参照 | ダイアログ閉じる + S-18 リスト再取得 + トースト |

```json
{
  "email": "sato@example.com",
  "password": "initial-password",
  "display_name": "佐藤花子",
  "role": "editor"
}
```

### 編集モード

| トリガー | メソッド | パス | リクエスト | レスポンス処理 |
|---|---|---|---|---|
| 保存ボタンクリック | PATCH | `/api/v1/users/:id` | 変更されたフィールドのみ | ダイアログ閉じる + S-18 リスト再取得 + トースト |

```json
{
  "role": "admin",
  "display_name": "佐藤花子（リーダー）"
}
```

### パスワードリセット

| トリガー | メソッド | パス | リクエスト | レスポンス処理 |
|---|---|---|---|---|
| リセット実行ボタンクリック | POST | `/api/v1/users/:id/reset-password` | `{ "new_password": "..." }` | トースト + リセットフォーム閉じる |

## エラー状態

| エラー | HTTP | 表示方法 |
|---|---|---|
| メールアドレス重複 | 422 | インラインエラー「このメールアドレスは既に使用されています」 |
| パスワードポリシー不適合 | 422 | インラインエラー「パスワードは 8 文字以上で入力してください」 |
| 認証切れ | 401 | ログイン画面（S-02）にリダイレクト |
| ネットワークエラー | — | ダイアログ内にインラインエラー |

## トースト・通知メッセージ

| 条件 | 種別 | メッセージ例 |
|---|---|---|
| ユーザー作成成功 | success | 「ユーザー「{display_name}」を追加しました」 |
| ロール変更成功 | success | 「{display_name} のロールを {role} に変更しました」 |
| パスワードリセット成功 | success | 「{display_name} のパスワードをリセットしました」 |
| 作成失敗（重複等） | error | 「ユーザーの追加に失敗しました」 |
